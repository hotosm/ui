üîß How It Works

The HOT UI library automatically injects CSS variables into shadow DOM elements so they can access HOT colors:

// Automatic injection when components are added to DOM import { setupAutoInjection, injectHOTThemeIntoAllComponents } from '@hotosm/ui'; // Set up auto-injection for new components setupAutoInjection(); // Manual injection for existing components injectHOTThemeIntoAllComponents();

üîç Inspecting Shadow DOM

Open your browser's developer tools and inspect a button element. You'll see:

<wa-button> #shadow-root (open) <style id="hot-theme-variables"> :host { --wa-color-brand-95: var(--hot-color-primary-50); --wa-color-brand-90: var(--hot-color-primary-100); /* ... more HOT color mappings ... */ } </style> <button part="base">...</button> </wa-button>

üìã Available CSS Variables

The following CSS variables are automatically available inside wa-button shadow DOM:

/* Brand Colors (mapped to HOT primary) */ --wa-color-brand-95 through --wa-color-brand-05 --wa-color-brand (default: --wa-color-brand-50) /* Neutral Colors (mapped to HOT gray) */ --wa-color-neutral-95 through --wa-color-neutral-05 --wa-color-neutral (default: --wa-color-neutral-40) /* Semantic Colors */ --wa-color-success-95 through --wa-color-success-05 --wa-color-warning-95 through --wa-color-warning-05 --wa-color-danger-95 through --wa-color-danger-05 /* Direct HOT Color Variables */ --hot-color-primary-950 through --hot-color-primary-50 --hot-color-success-950 through --hot-color-success-50 --hot-color-warning-950 through --hot-color-warning-50 --hot-color-danger-950 through --hot-color-danger-50 --hot-color-neutral-950 through --hot-color-neutral-50 /* HOT Typography */ --hot-font-sans, --hot-font-sans-variant, --hot-font-mono --hot-font-size-*, --hot-font-weight-* --hot-spacing-*, --hot-border-radius-*
